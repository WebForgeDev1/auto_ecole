Viewport Meta + Base CSS Mobile-First
+25 pts
Ajouter le viewport meta et définir une base CSS mobile-first avec typographie fluide.
📄 index.html (dans <head>)
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=5.0"> <meta name="theme-color" content="#0f172a">
📄 styles.css
/* Mobile-first base */ * { box-sizing: border-box; } html { font-size: 16px; } body { font-family: 'Inter', system-ui, sans-serif; line-height: 1.6; color: #1e293b; overflow-x: hidden; -webkit-font-smoothing: antialiased; } /* Fluid typography — pas de media queries intermédiaires */ h1 { font-size: clamp(1.5rem, 5vw, 2.5rem); } h2 { font-size: clamp(1.25rem, 4vw, 2rem); } h3 { font-size: clamp(1.1rem, 3vw, 1.5rem); } p, li { font-size: clamp(0.9375rem, 2.5vw, 1.0625rem); }
💡 Le clamp() assure une taille fluide entre mobile et desktop sans media queries intermédiaires.

Images Optimisées (WebP + Lazy + Srcset)
+15 pts
Convertir toutes les images en WebP avec srcset responsive et lazy loading.
<picture> <source srcset="img/hero-400.webp 400w, img/hero-800.webp 800w, img/hero-1200.webp 1200w" sizes="(max-width: 600px) 100vw, (max-width: 1024px) 80vw, 1200px" type="image/webp"> <img src="img/hero-800.jpg" alt="Moniteur d'auto-école Hk Conduite au Mans" loading="lazy" decoding="async" width="800" height="600"> </picture>

🛠️ Utiliser Squoosh.app ou Sharp (Node.js) pour convertir en WebP. Objectif : < 100KB par image.
